IL NOSTRO PROGETTO

L'obiettivo del progetto è mostrare una delle moltissime applicazioni della costante aurea (costante di fibonacci). In particolare questo progetto riguarda l'ambito dell'informatica.
In informatica la costante aurea è utilizzata per creare deglinalgoritmi di hashing molto efficienti. Vediamo nello specifico cosa significa.

COSA È UN HASH

Un hash è un tipo di algoritmo pseudorandomico che prende un input di lunghezza arbitraria e restituisce una stringa di lunghezza fissa.
Un algoritmo di hashing deve avere quattro caratteristiche:

  • Unidirezionale
  • Deterministico
  • Caotico
  • Resistente alle collisioni
Vediamo più nello specifico cosa significa.

CARATTERISTICHE


Deve essere unidirezionale: data una stringa S0, risultato dell'operazione di hashing su un dato S1, non ci deve essere modo di risalire a S1 anche conoscendo S0 e l'algoritmo di hashing.
Deterministico vuol dire che dato lo stesso input l'algoritmo deve restituire lo stesso output, ma deve allo stesso tempo minimizzare le collisioni, cioè la probabilità che due input diversi diano lo stesso output.
Infine la qualità più importante di un algoritmo di hashing è l' "imprevedibilità" del'output.
Per prendere in prestito un termine dalla fisica un sistema "caotico" è un sistema in cui anche il cambiamento più piccolo nelle condizioni iniziali cambia completamente il risultato. Allo stesso modo in un buon hash il cambiamento anche di un solo bit nell'input cambia completamente l'ouput.


UTILIZZO

Le funzioni di hashing hanno due utilizzi principali:

  • Crittografia
  • Hash tables

Per essere coincisi possiamo dire che vengono usate in crittografia per la verifica dell'autenticità e la firma digitale.
Le hash tables sono invece un modo di organizzare dati con delle coppie chiave-valore in cui l'hash della chiave corrisponde alla posizione del valore nella hashtable. Questo permette di cercare dati con una complessità temporale media di O(1).

FIBONACCI HASHING

Per "Fibonacci hashing" si intende un tipo di hashing in cui la funzione usa la costante aurea (derivata dalla sequenza di fibonacci) per "randomizzare" l'input. In particolare il nostro algoritmo usa una funzione di multiplicative hashing di tipo:

h(k) = M * (k*A % 1)

Dove:
h(k): l'hash della chiave.
M: la dimensione dell'array.
k: la chiave (int).
A: Una costante (costante aurea in questo caso).
